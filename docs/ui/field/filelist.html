Filelist

import Filelist from '@stackpress/ink-ui/field/filelist';

Props

The `<field-filelist>` component allows uploading and managing multiple files, displaying them as a list with optional previews. Here’s how each prop shapes its behavior:

Property Type Required Description name String No Sets the name attribute for hidden `<input>` elements, one per uploaded URL, enabling form submission of multiple files. image Boolean No Restricts the input to image files (`accept="image/*"`) and shows a 32px-high preview for each uploaded image. uploading String No Text shown when files are being uploaded. Defaults to "Uploading...". Customize it for clarity or localization. value Array No Initial array of file URLs to display. Defaults to an empty array. Each URL gets a link and optional preview. change Function No Triggered when files are selected, receiving the change event. Useful for pre-upload processing or logging. upload Function No Handles uploading multiple files. Receives an array of files and a callback to return their URLs. Essential for upload functionality. class String No Custom CSS classes for the host element, allowing layout tweaks like width or spacing. style String No Inline styles for the host element, overriding defaults for precise visual control.

Note: The `update` prop exists but is unused in the current implementation. It’s included for potential future enhancements.

Basic Filelist Input

A simple multi-file input with a form name. It accepts any file type and stores URLs in hidden inputs for form submission.
<field-filelist name="files" />

Image-Only Filelist

Limits selection to images and previews them after upload. The `image` prop ensures only image files are accepted.
<field-filelist name="files" image />

Filelist with Initial URLs

Displays a pre-populated list of URLs using the `value` prop. Each URL has a link and preview (if `image`), with a remove option.
<field-filelist name="files" image value={["https://example.com/image1.jpg", "https://example.com/image2.jpg"]} />

Filelist with Custom Uploading Text

Shows custom text during upload via `uploading`. Pair with `upload` to see it while files process (simulated here).
<script> const handleUpload = (files, callback) => { setTimeout(() => callback(files.map(f => "https://example.com/uploads/" + f.name)), 1000); }; </script> <field-filelist name="files" uploading="Processing files..." upload={handleUpload} />

Filelist with Change Event

Logs selected file names with the `change` prop. Great for pre-upload validation or user feedback.
<script> const handleChange = (e) => console.log('Selected files:', Array.from(e.target.files).map(f => f.name)); </script> <field-filelist name="files" change={handleChange} />

Filelist with Upload Handler

Uses `upload` to process multiple files, adding their URLs to the list after a simulated 1-second delay.
<script> const handleUpload = (files, callback) => { setTimeout(() => callback(files.map(f => "https://example.com/uploads/" + f.name)), 1000); }; </script> <field-filelist name="files" upload={handleUpload} />

Styled Filelist Input

Customizes the file list with a wider layout, rounded corners, and a vibrant border using `class` and `style`. Enhances visibility for multiple files.
<field-filelist name="files" class="w-400 bg-t-0 rounded-10" style="border: 3px solid #ff4081; padding: 10px;" />
