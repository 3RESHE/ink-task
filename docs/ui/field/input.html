Input

import Input from '@stackpress/ink-ui/field/input';

Props

The `<field-input>` component is a flexible wrapper for native `<input>` elements, supporting various types and attributes. Here’s how its props work:

Property Type Required Description type String No Sets the input type (e.g., "text", "file", "checkbox", "number"). Defaults to "text" if unset. name String No Name attribute for form submission, tying the input to form data. value String No Initial value of the input. For "checkbox" or "radio", use "checked" instead. placeholder String No Hint text shown when the input is empty, guiding user input. accept String No File types allowed for "file" inputs (e.g., "image/*"). Ignored for other types. autocomplete String No Controls browser autofill behavior (e.g., "on", "off", "email"). checked Boolean No Sets initial checked state for "checkbox" or "radio" inputs. disabled Boolean No Disables the input, preventing interaction. max String/Number No Maximum value for numeric or date inputs (e.g., "100"). min String/Number No Minimum value for numeric or date inputs (e.g., "0"). multiple Boolean No Allows multiple file selections for "file" inputs. pattern String No Regex pattern for input validation (e.g., "[0-9]{3}"). readonly Boolean No Makes the input read-only, allowing viewing but not editing. required Boolean No Marks the input as required for form submission. step String/Number No Increment step for numeric inputs (e.g., "0.1"). change Function No Custom handler for change events, receiving the event object. update Function No Callback with the current input value after changes. class String No CSS classes for the host element, styled via default utilities. style String No Inline styles for the host element, overriding defaults.

Basic Text Input

A simple text input with a name and placeholder, ideal for basic form fields like usernames or comments.
<field-input name="text" placeholder="Enter text" />

File Input

A file input with `accept` and `multiple`, allowing selection of multiple images for upload.
<field-input type="file" name="files" accept="image/*" multiple />

Checkbox Input

A checkbox with an initial checked state, useful for toggles or boolean form options.
<field-input type="checkbox" name="agree" checked />

Number Input

A numeric input with `min`, `max`, and `step`, perfect for controlled ranges like quantities.
<field-input type="number" name="quantity" min="0" max="100" step="5" value="10" />

Input with Validation

A text input with `pattern` and `required`, enforcing a three-digit format (e.g., "123").
<field-input type="text" name="code" pattern="[0-9]{3}" required placeholder="Enter 3 digits" />

Input with Change Event

Logs changes with the `change` prop, capturing user input for real-time feedback or validation.
<script> const handleChange = (e) => console.log('Change event:', e.target.value); </script> <field-input name="text" change={handleChange} placeholder="Type here" />

Input with Update Callback

Uses `update` to track the current value, ideal for syncing state or triggering actions.
<script> const handleUpdate = (value) => console.log('Updated value:', value); </script> <field-input name="text" update={handleUpdate} placeholder="Type here" />

Styled Input

A minimalist styled input with a wide layout, soft border, and hover effect via `class` and `style`.
<field-input name="text" class="w-300 bg-white tx-t-1 rounded-5" style="border: 1px solid #e0e0e0; padding: 8px; transition: border-color 0.3s ease;" onmouseover="this.style.borderColor='#888888'" onmouseout="this.style.borderColor='#e0e0e0'" placeholder="Enter text" />
