File

import File from '@stackpress/ink-ui/field/file';

Props

The `<field-file>` component provides a file upload interface with states for selection, uploading, and display. Below are its props and their roles:

Property Type Required Description name String No Sets the name attribute for a hidden `<input>` that stores the uploaded file URL, integrating with forms. image Boolean No When present, restricts the file input to image types (sets `accept="image/*"`) and displays a preview of the uploaded image. uploading String No Custom text displayed while a file is uploading. Defaults to "Uploading...". Useful for localization or branding. value String No Initial URL of a file, bypassing the upload step. Displays the file link and optional image preview. change Function No Called when a file is selected, receiving the change event. Useful for tracking file selection before upload. update Function No Callback invoked with the uploaded file URL after a successful upload. Ideal for updating state or triggering actions. upload Function No Handles file upload logic. Takes the selected file and a callback to set the URL. Required for uploading functionality. class String No Custom CSS classes applied to the component’s host element, allowing layout and style adjustments. style String No Inline styles applied to the host element, overriding default styles for precise control.

Basic File Input

A minimal file input with a form name, ready to accept any file type and store its URL in a hidden input once uploaded.
<field-file name="file" />

Image-Only File Input

Restricts the input to image files and previews the uploaded image. The `image` prop ensures only images are selectable.
<field-file name="file" image />

File with Initial URL

Displays an existing file URL with a clickable link and reset option. The `value` prop sets the initial state, skipping upload.
<field-file name="file" value="https://images.wsj.net/im-580612/8SR" image />

File with Custom Uploading Text

Customizes the text shown during upload with the `uploading` prop. Pair with `upload` to see it in action (simulated here).
<script> const handleUpload = (file, callback) => { setTimeout(() => callback("https://example.com/uploads/" + file.name), 1000); }; </script> <field-file name="file" uploading="Processing your file..." upload={handleUpload} />

File with Change Event

Triggers a `change` handler when a file is selected, logging the file name. Useful for pre-upload validation or feedback.
<script> const handleChange = (e) => console.log('Selected file:', e.target.files[0]?.name); </script> <field-file name="file" change={handleChange} />

File with Update Callback

Calls the `update` function with the uploaded URL after upload completes. Requires `upload` to process the file.
<script> const handleUpdate = (url) => console.log('Uploaded URL:', url); const handleUpload = (file, callback) => { setTimeout(() => callback("https://example.com/uploads/" + file.name), 1000); }; </script> <field-file name="file" update={handleUpdate} upload={handleUpload} />

File with Upload Handler

Uses the `upload` prop to handle file uploads, simulating a 1-second delay before returning a URL.
<script> const handleUpload = (file, callback) => { setTimeout(() => callback("https://example.com/uploads/" + file.name), 1000); }; </script> <field-file name="file" upload={handleUpload} />

Styled File Input

Customizes appearance with `class` and `style`, adjusting width and border for a unique look.
<field-file name="file" class="w-300" style="border: 2px dashed #007bff;" />
